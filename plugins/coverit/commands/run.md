# Coverit Run

Execute test files that were generated by a prior `/coverit:generate`. This is the final step in the SGR pipeline (Scan → Generate → Run).

Does NOT generate or analyze — only executes existing test files and reports results.

## Arguments

Parse from user input:
- `[runId]` - Run ID from a prior generate (defaults to latest run). Anything starting with `run-` is a runId.
- `[path]` - Project root path (defaults to current working directory)
- `--plan <ids>` - Comma-separated plan IDs to run (default: all plans with test files)

## Execution

### Phase 1: Run via Sub-Agent

Spawn a sub-agent using the Task tool with `subagent_type: "general-purpose"` and `run_in_background: true`.

```
Call the `mcp__plugin_coverit_coverit__coverit_recheck` MCP tool with:
{
  "projectRoot": "<absolute path>",
  "runId": "<runId>",           // only include if user specified a runId
  "planIds": ["<planId>", ...]  // only include if user specified --plan
}

Return a concise summary (NOT the full JSON):
- For each plan: planId, status (passed/failed), tests passed/failed, duration, test file path
- Overall: N/M plans passing, total tests, total duration
```

**CRITICAL**: The sub-agent MUST use the `mcp__plugin_coverit_coverit__coverit_recheck` MCP tool. It must NOT use the CLI, shell commands, or attempt to run tests manually.

### Phase 2: Report Results

Wait for the sub-agent to complete, then show the summary:

```
Run complete for <runId>.

Results: N/M plans passing

  plan_001: passed (5/5 tests, 1.2s) — src/utils/hash.test.ts
  plan_002: failed (3/5 tests, 2.1s) — src/services/auth.test.ts
  ...

Total: X tests passed, Y failed (Zs)
```

If all tests pass: "All tests pass."

If any tests failed: "Run `/coverit:fix` to attempt AI-powered fixes, or edit the test files manually and run `/coverit:check` to re-verify."

If no test files found: "No generated test files found. Run `/coverit:generate` first, or use `/coverit:full` to do everything at once."
